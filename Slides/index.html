Lesson 3: Filtering and Sorting Data

What we will cover:

  2. ko.utils.arrayFilter
  3. ko.utils.arrayFirst

ko.Utils - What is it?

Internally, Knockout has a number of utility functions used by the library itself, but many of these functions might actually be handy for anyone using the library.

ko.utils.arrayFilter

We bind a textbox to a filter observable and use it to get our filtered items

              //filter the items using the filter text
viewModel.filteredItems = ko.computed(function() {

    var filter = this.filter().toLowerCase();

    //Check if any filter has been defined
    if (!filter) {

        return this.items();

    } else {

        //Filter the array with the given filter string
        return ko.utils.arrayFilter(this.items(), function(item) {
            return ko.utils.stringStartsWith(item.name().toLowerCase(), filter);
        });
    }
}, viewModel);

ko.utils.arrayFirst

Return the first item that matches


//identify the first matching item by name
viewModel.firstMatch = ko.computed(function() {

    var search = this.search().toLowerCase();

    //check if there is a search value
    if (!search) {

        return null;

    } else {

        //Return the first item in the array that matches
        return ko.utils.arrayFirst(this.filteredItems(), function(item) {

            return ko.utils.stringStartsWith(item.name().toLowerCase(), search);

        });
    }
}, viewModel);

ko.utils.arrayForeach

Short hand to loop through array


    //Function to compute the total of an array of items
    viewModel.total = ko.computed(function() {

        var total = 0;

        //Loop through given array
        ko.utils.arrayForEach(this.items(), function(item) {

            //Sumup total
            var value = parseFloat(item.priceWithTax());
            if (!isNaN(value)) {
                total += value;
            }

        });

        return total.toFixed(2);
    }, viewModel);

ko.utils.arrayMap

ko.utils.arrayMap is a nice way to take an array of objects and generate a flattened structure.


              //get a list of used categories
              viewModel.justCategories = ko.computed(function() {

                  var categories = ko.utils.arrayMap(this.items(), function(item) {
                      return item.category();
                  });

                  return categories.sort();

              }, viewModel);

Welcome to AngularJS


What is Angularjs?

A MVVM framework that provides data binding, routing, and APIs for connecting to server-side code.

Angularjs is designed to encourage structuring your application is a loosely coupled manner by providing built in support for dependency injection and strong support for unit testing.


Who are the authors?

A couple of guys from Google.

Lesson 1: An app with a view


What we will cover:


  1. Bootstrapping our app using the ng-app directive.
  2. Declaring a simple controller to bind data to the view.
  3. Using the $scope variable as our view-model.

Displaying data using Angular


  1. Add data to the $scope variable.
  2. Tell Angular where to display the data in the markup by using double brace markers '{{}}'.
  3. Angular automatically replaces the {{}} with values from the $scope as required.

Binding is two way, i.e. edits to the bound data values within the markup will be reflected back to the $scope variable.

Bootstrapping our Angular app


Declaring our application scope

The ng-app directive is used to tell Angular where the scope of our application begins. It is normal to add the directive as an attribute of either the <html> or <body> elements. 


              <html ng-app="myApp">

Adding a Controller


Controllers are pieces of executable JavaScript code that are responsible for associating data with the current scope of a view.

Controllers should retrieve the data to be displayed from your model, be it a web services or other data source, and add elements to the $scope variable that need to be made available to the view. 


              <body ng-controller="MyController">

The ng-controller directive tells Angular where to apply a controller within your markup. Child elements of the element containing the ng-controller directive will have access to the $scope associated with your controller for binding purposes.

Wiring up the JavaScript code


With the directives in place we can now add the code that executes at runtime when Angular processes our page. First we get a reference to our application by calling the module() function of the global angular reference. 


              var myApp = angular.module('myApp', []);

Wiring up the JavaScript code (2)


Then we declare our controller calling the controller() function on the application reference passing in a function that accepts the $scope argument. 


                myApp.controller('MyController', function($scope) {
                  // Add awesome things to the $scope
                  $scope.awesome_thing = 'OMG so awesome!';
                });

Now we can bind data from $scope to our view


Items from the $scope are reference in the view using double braces '{{}}' to surround the name of the $scope attribute to reference. 


              <p>This value is from the $scope: {{awesome_thing}}<p>

Repeat all the data!


We can then go wild with ng-repeat and bind to arrays of data! 


              myApp.controller('MyController', function($scope) {
                $scope.array_of_things = [
                  { name: 'Chris', occupation: 'Ninja' },
                  { name: 'David', occupation: 'Rockstar' },
                  { name: 'Michael', occupation: 'Meerkat Tamer' }
                ];
              });

              <div ng-repeat="thing in array_of_things">
                <label>Name:</label> {{thing.name}}
                <label>Occupation:</label> {{thing.occupation}}
              </div>

Making HTTP requests for data


AngularJS provides an $http function that can be called to make an HTTP request to a remote server.

$http takes a method and a URL and returns a promise that has two functions that handle the result of the request: 


$http({method: 'GET', url: 'http://localhost:3030/getMeerkats'})
  .success(function(data, status, headers, config) {
    // Read the data here.
  })
  .error(function(data, status, headers, config) {
    // Handle errors here.
  });

Binding the returned data to our view



$http({method: 'GET', url: 'http://localhost:3030/getMeerkats'})
  .success(function(data, status, headers, config) {
    // All we need to do is add the data to our $scope and AngularJS will
    // do the rest :)
    $scope.meerkats = data;
  });

AngularJS will automatically update the DOM with the new data added to the $scope.
